---
name: update-docs
description: Add or update project documentation beyond the README — architecture notes, API references, workflow docs. Use when a change introduces or modifies a feature, API, or architectural decision that needs documentation beyond the README.
---

1. Check whether existing docs cover the affected area; update in place if so, create a new page under the project's docs folder if not.
2. Keep docs organized (consistent folder/naming convention) and cross-linked rather than duplicating README content.
3. Apply `DOC-001`: document non-obvious why and how, not a restatement of the code.
4. Link the new or updated doc from the README or an index page if one exists, so it's discoverable.

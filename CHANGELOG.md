## [Unreleased]

## [1.1.0] - 2026-07-19

### Added

- Self-hosted the Turret Road font instead of loading it from the Google
  Fonts CDN, removing a third-party render-blocking request and avoiding
  leaking visitor IPs to Google on every page load.
- Confirmation dialog before resetting the high score, replacing the
  browser's default `confirm()` prompt with an in-page `<dialog>` styled
  to match the game's theme.
- `Content-Security-Policy` meta tag (`default-src 'self'`) hardening the
  served page.
- CI workflow that runs `npm ci` and `npm run build` on push and pull
  request.
- Release workflow that packages `dist/` as a downloadable zip attached to
  a GitHub Release on `v*` tag pushes, additive to the existing gh-pages
  deploy.
- Vitest unit test suite (26 tests) covering the pure-logic model classes
  (`Entity`, `Asteroid`, `Bullet`, `Difficulty`, `Spaceship`), replacing the
  always-failing `npm test` stub.
- ESLint flat config (`eslint.config.js`) with `eslint-config-prettier`, a
  committed `.prettierrc` locking in the previously-implicit formatting
  defaults, and `.editorconfig`.
- Lint, format-check, and test steps wired into CI ahead of the build step.
- Generic entity draw interface: `Canvas#drawEntity` dispatches on
  `entity.getDrawShape()` instead of a per-type `drawSpaceship`/
  `drawAsteroid`/`drawBullet` method, so new entity types won't need their
  own `Canvas` draw method.

### Changed

- Migrated build tooling from webpack to Vite, clearing all known
  `npm audit` vulnerabilities at the root instead of patching them.
  `dist/` is now fully generated by `npm run build` from a root
  `index.html` and `public/` instead of being hand-maintained and
  committed.
- Cached the `localStorage` high-score read in a JS variable instead of
  re-reading it up to twice per frame, only writing back when a new high
  score is actually set.
- Scaled the canvas backing buffer by `devicePixelRatio` while keeping the
  CSS-visible size at the intended 800x600 layout, so rendering is sharp
  on high-DPI displays.
- Reorganized the model layer: moved steering (`jerk`, `maxD`,
  accelerate/decelerate) from `Entity` down to `Spaceship`, the only
  subclass that steers, and deduped the max-speed clamp into
  `clampSpeed()`. Renamed `Entity.mathifyOrientation` to
  `orientationToRadians`.
- Consolidated HUD drawing: `drawScore`/`drawHigh`/`drawPaused` collapsed
  into a single `drawText`, and `drawDiff` into `drawSelector` + `drawText`.
- Moved `sound.js` from `src/models/` to `src/lib/`, alongside `canvas.js`,
  since both are stateful wrappers around a browser API rather than domain
  models.

### Fixed

- Array-mutation-during-iteration bugs in the bullet/asteroid collision
  and bullet-expiry loops.
- Rapid-fire laser sound replay bug.
- Missed-keypress bug in pause/confirm handling by latching keydown edges
  per frame instead of sampling held state.
- `Math.log(0)` asteroid-spawn guard.
- `pauseGame()` message bug where the paused overlay could show stale
  text depending on call site.
- `Asteroid.getBounds()` returning coordinates relative to its own center
  instead of absolute coordinates, inconsistent with `Bullet`/`Spaceship`.
- `canvas.js` hardcoding `"#FFFFFF"` for asteroids/bullets instead of
  reading `entity.color`, so entity color is now genuinely configurable.
- An unused variable and dead zero-initializers in `spawnAsteroid`
  (`src/index.js`), surfaced by the new ESLint config.

### Removed

- Dead code: the unreachable, syntactically broken `src/controllers/asteroids.js`
  duplicate and the unused `src/lib/GameLib/` tree.
- Dead `getX()`/`getY()` accessors from `Entity` in favor of direct
  property access, matching the rest of the codebase's convention.
